# 🍃 基于 Spring Boot 的茶叶溯源销售与文化展示系统

本项目是一个集**茶叶在线销售**、**全流程生产溯源**、**茶文化科普**以及**后台管理系统**于一体的综合性 Web 平台。系统采用前后端分离的架构思路，旨在解决传统茶叶交易中信息不透明、品质核验难的问题。

## ✨ 核心功能

### 👤 用户端

* **注册登录**：包含验证码校验及角色初始化（普通用户、测试用户）。
* **商品展示**：支持按分类筛选、关键词搜索及价格/库存显示。
* **全流程溯源**：通过溯源码查询茶叶的产地环境、种植采摘、工艺加工及检测报告。
* **模拟支付**：内置沙箱模拟支付功能，支持用户余额扣款验证交易闭环。
* **个人中心**：管理个人资产（余额）、查看历史订单记录。

### 🛠 管理员端 (Admin)

* **数据可视化**：基于 ECharts 的后台看板，实时监控总销量、商品总数及用户活跃趋势。
* **商品管理**：支持茶叶信息的发布、编辑、删除以及商品图片的上传管理。
* **轮播配置**：动态管理首页轮播图，支持图片上传与排序。
* **订单处理**：全站订单状态监控及发货管理。
* **用户管理**：查看全站用户列表、权限标签及账户余额。

## 🚀 技术栈

* **后端**: Spring Boot 3, MyBatis Plus
* **前端**: Vue 3, Element Plus, Axios
* **数据库**: MySQL 8.0
* **可视化**: ECharts
* **验证码**: Easy Captcha

## 🛠️ 本地快速部署

### 1. 环境要求

* JDK 17+
* Maven 3.6+
* MySQL 8.0+

### 2. 数据库配置

1. 创建名为 `tea_db` 的数据库。
2. 执行项目中的数据库初始化 SQL 脚本。
3. 修改 `src/main/resources/application.yaml` 中的数据库连接信息（用户名及密码）。

### 3. 图片资源说明

由于项目支持图片上传功能，手动迁移项目时，请务必保证图片文件夹的完整性：

* **路径**：`src/main/resources/static/uploads/`
* **Git 提示**：若通过 GitHub 克隆后无法显示图片，请检查该目录是否被包含。本地上传的新图会自动保存在此位置。

### 4. 运行项目

1. 使用 IDEA 打开本项目。
2. 运行 `TeaTraceSystemApplication.java`。
3. 浏览器访问：`http://localhost:8080/index.html`。

## 📂 仓库目录结构

```text
├── java/com/tea/trace
│   ├── controller/      # API 接口逻辑 (Auth, Tea, File, etc.)
│   ├── entity/          # 数据库实体类 (User, Product, Order, Banner)
│   ├── mapper/          # MyBatis Mapper 接口
│   └── service/         # 业务逻辑层
├── resources/
│   ├── static/          # 前端页面 (index.html, profile.html, uploads/)
│   └── application.yaml # 系统配置文件
└── pom.xml              # Maven 依赖配置

```

## 📝 开发备注

* **沙箱支付**：登录角色为 `TEST_USER` 或 `ADMIN` 时可体验完整支付流程，支付金额将从账户余额中扣除。
* **溯源逻辑**：每件商品生成的 `traceCode` 是唯一的，对应其在生产环境中的全生命周期数据。
